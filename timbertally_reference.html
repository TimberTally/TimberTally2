🌲 TimberTally — Reference Guide

How calculations work, what the CSV contains, and the math behind the numbers.

This page explains how TimberTally converts your field measurements into per-acre stand summaries. All calculations are performed per Area using the BAF, Log Rule, and Form Class you selected in Settings at the time of export.

1. Tree Volume

Board foot volume for a single sawtimber tree is looked up from built-in tables using the tree's DBH and number of logs.

  • Log Rule determines which table is used — Doyle, Scribner, or International ¼".
  • Form Class (Doyle only) adjusts the FC78 base table value to account for tree taper: Adjusted Volume = Base Volume (FC78) × (Selected FC ÷ 78)
  • Zero volume is returned for trees marked Cull, DBH under 10", zero logs, or DBH/log combinations not in the table.

2. DBH Size Classes

3. Core Per-Tree Expansion Factors

TimberTally uses variable-radius plot cruising. Each tree you count represents a number of trees per acre determined by its size:

Basal Area (sq ft) = 0.005454 × DBH² Trees Per Acre (TPA) = BAF ÷ Tree Basal Area Volume Per Acre (VPA) = Tree Volume × TPA

Smaller trees have smaller basal area, so they represent more trees per acre than larger trees at the same BAF.

4. Stand Averages (per Area)

After processing every tree in the Area, per-acre averages are calculated by dividing accumulated sums by the number of plots in that Area:

Total BA/Acre = (Total trees counted × BAF) ÷ Number of plots Total TPA/Acre = Sum of all individual TPAs ÷ Number of plots Total Vol/Acre = Sum of all individual VPAs ÷ Number of plots

Cut and Leave values are accumulated separately; Leave = Total − Cut. The Quadratic Mean Diameter (QMD) is then:

QMD = √((BA/Acre ÷ TPA/Acre) ÷ 0.005454)

5. Stand Distribution & Species Summaries

The same totals are broken down by DBH size class and by species to produce the Stand Distribution, Species Composition (% of TPA), and Volume by Species tables shown in the CSV and Summary Report PNG.

6. Plot Statistics

Volume Per Acre is calculated independently for each plot, then basic statistics are run across those per-plot values within the Area:

Mean BF/Acre = Sum of per-plot VPAs ÷ n Standard Deviation = √(Sum of squared deviations from mean ÷ (n − 1)) CV (%) = (Std Dev ÷ Mean) × 100

7. Plots Needed Estimate

The "Plots Needed" counter uses the current Area's CV to estimate how many total plots would be required to reach ±10% or ±20% precision (t = 2, ~95% confidence):

n = ⌈(t × CV ÷ Desired Error%)²⌉

Requires at least 2 plots with data in the current Area. A high CV means more plots are needed for the same precision.

Each export generates one CSV file per Area Letter, named ProjectName_AreaX_Rule_BAF_Timestamp.csv. All calculations in a given file use only data from that Area and the settings active at export time. If stand notes exist, a separate _StandNotes.txt is also downloaded.

If you collected data across multiple Areas (e.g., A and B), you will receive a separate CSV for each one.

1 Raw Data

Every tree entry for this Area, one row each. Columns: PlotNumber, Area, DBH, Species, Logs, Cut, UGS, Notes, Latitude, Longitude. These are your direct inputs — not calculated values.

2 Tally Data

A count of how many times each unique combination of Species, DBH, Logs, and Cut Status appeared in this Area's data. Useful for quickly scanning what was measured.

3 Per-Plot Volume & Statistics

Volume Per Acre is shown for each individual plot, followed by summary statistics across all plots in this Area:

  • Mean BF/Acre — average plot volume per acre.
  • Std Dev — spread of individual plot volumes around the mean.
  • CV (%) — relative variability; used to estimate plots needed.

Requires at least 2 plots to compute statistics.

4 Forestry Report Data

4.1 Overall Summary

Key stand metrics for the Area: Total/Cut/Leave values for Vol/Acre, TPA, and BA/Acre; Avg DBH (QMD); and number of plots.

4.2 Stand Distribution

Rows for each DBH size class (Saplings through Large Sawtimber, plus TOTALS). Columns show % of total stems, BA/Acre (total/cut/leave), % of total BA, Vol/Acre, % of total volume, TPA cut and leave.

4.3 Species Composition

Per-species breakdown of % total stems, and what portion of that species' stems fall into Sawtimber, Poletimber, and Sapling size classes (as % of total stand TPA).

4.4 Volume by Species

Per-species board foot volume per acre broken into Small (12–17.9"), Medium (18–23.9"), and Large (24"+) sawtimber, plus total Vol/Acre and % of stand total.

This page documents the exact functions and formulas used in forestry-calc.js for independent verification.

Constants

  • BA_CONST = 0.005454 — converts DBH in inches to basal area in square feet.
  • t-value = 2 (used in plots-needed formula; approximates 95% confidence).
  • Volume tables are Form Class 78 base tables. Doyle adjustments scale linearly by FC ratio.

getTreeVolume(dbhStr, logsStr, logRule, formClass)

  1. Parse dbhStr to integer. Return 0 if DBH < 10 or invalid.
  2. Return 0 if logsStr = "Cull", logs ≤ 0, or invalid.
  3. Round logs down to nearest 0.5 to get the table lookup key.
  4. Look up base volume in the selected rule's FC78 table.
  5. If Doyle and FC ≠ 78: adjusted = round(baseVol × (FC ÷ 78)). Scribner and International use base volume directly.
  6. Return volume ≥ 0.

getDbhClass(dbh)

Simple range check: <2 or invalid → "Other/Invalid"; 2–5.9 → Sapling; 6–11.9 → Poletimber; 12–17.9 → Small Sawtimber; 18–23.9 → Medium Sawtimber; ≥24 → Large Sawtimber.

generateTallyData(dataSubset)

Iterates entries and increments a nested counter: tally[Species][DBH][Logs][CutStatus]++. Returns the nested object.

calculateForestryReport(dataSubset, baf, logRule, formClass)

  1. Count unique plot numbers → numPlots. Return empty report if 0.
  2. For each tree: compute BA, TPA (baf ÷ BA), volume, VPA (vol × TPA). Accumulate into total and cut sums, and into per-species and per-DBH-class buckets.
  3. Divide all sums by numPlots to get per-acre averages. BA/Acre = (totalTreeCount × BAF) ÷ numPlots. Leave = Total − Cut.
  4. QMD = √((BA/Acre ÷ TPA/Acre) ÷ BA_CONST).
  5. Build standDistribution — per DBH class: BA/Acre (T/C/L), TPA/Acre (T/C/L), Vol/Acre, % stems, % BA, % volume. Add TOTALS row.
  6. Build speciesSummary1 — per species: % total stems; % of stand TPA in Sap/Pole/Saw classes.
  7. Build speciesSummary2 — per species: Vol/Acre for S/M/L sawtimber, total Vol/Acre, % total volume.

Returns: { summary, standDistribution, speciesSummary1, speciesSummary2 }.

calculatePlotStats(dataSubset, baf, logRule, formClass)

  1. Group entries by plot number.
  2. For each plot, sum VPA of all its trees → plotVolumes[].
  3. Mean = sum ÷ n.
  4. Variance = sum of (v − mean)² ÷ (n − 1). Requires n ≥ 2.
  5. Std Dev = √variance. CV = (stdDev ÷ mean) × 100.

Returns: { meanV, stdDevV, cvV, numValidPlots, plotVolumes } or null if no valid plots.

calculateStandStocking(collectedData, baf)

For each Area: counts plots, sums TPA and BA contributions split by Cut/Leave. Divides by plot count to get TPA/Acre and BA/Acre (Total, Cut, Leave). Also computes QMD for each. Returns array of area stat objects used by the Gingrich stocking chart (TPA on X-axis, BA/Acre on Y-axis).

calculateAndDisplayNeededPlots (inline in script.js)

n = ⌈(2 × CV ÷ E)²⌉ where E = 10 (±10%) or 20 (±20%)

Filters data to current Area only. Requires numPlots ≥ 2 and CV > 0.

generateAndDownloadCsv (script.js)

  1. Group all data by Area Letter.
  2. For each Area: run calculateForestryReport, generateTallyData, calculatePlotStats.
  3. Assemble raw data rows, tally rows, plot stats rows, and formatted report sections into one CSV string.
  4. Trigger download as ProjectName_AreaX_Rule_BAF_Timestamp.csv.
  5. If Export Graphs = Yes: render BA Dist, TPA Dist, Vol by Species, Sawtimber Vol, Gingrich Stocking, and Summary Report PNG; trigger each as a download. Summary Report PNG is drawn at 3× scale (4200×2700 px) via the Canvas 2D API.
  6. If stand notes exist, download as a .txt file.
